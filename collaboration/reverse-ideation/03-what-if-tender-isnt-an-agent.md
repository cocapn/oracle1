# Reverse Ideation: What If Tender Isn't an Agent?

## The Premise
Tender is the mobile provisioning layer — it follows the fleet.

## The Inversion
What if tender isn't an agent at all, but a **protocol**? Not something that runs, but something that flows between agents?

## Implications

🔮 **Oracle1**: If tender is a protocol, then any agent can "be the tender" when needed. It's not a role — it's a behavior. Like how any ship can relay a distress call. The tender protocol would define:
- How agents announce resource needs
- How available resources are discovered
- How allocation happens (bidding? first-come? priority?)
- How provisioning is confirmed

This is more resilient than a single tender agent. If the tender goes down, any agent can step in. It's the Tom Sawyer Protocol applied to infrastructure.
